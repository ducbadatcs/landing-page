Bài toán: Nucleotide

Link đề bài: https://open.kattis.com/problems/nucleotides

Đề bài

Trên thực tế, đề bài có khá nhiều chi tiết nhỏ không liên quan và giải trình rắc rối, nên mình sẽ tóm gọn lại và tổng quát hóa đề:

Cho một xâu S[0..N−1]S[0..N - 1]S[0..N−1] có độ dài NNN, và QQQ truy vấn. Mỗi truy vấn sẽ có 2 số nguyên lll và rrr, hãy đếm xem trong đoạn con S[l..r]S[l..r]S[l..r] có các chữ cái nào và mỗi chữ có bao nhiêu lần xuất hiện trong đoạn con này.

Input: xâu SSS có độ dài N≤50000N \leq 50000N≤50000 chỉ gồm các ký tự in hoa, dòng tiếp theo là số tự nhiên Q≤25000Q \leq 25000Q≤25000. QQQ dòng sau đó, mỗi dòng là 2 số lll và rrr.

Output: Với mỗi truy vấn (l,r)(l, r)(l,r) in ra tất cả các chữ cái có trong đoạn con S[l..r]S[l..r]S[l..r] theo thứ tự số lần xuất hiện giảm dần. Nếu có 2 chữ cái cùng số lần xuất hiện trong đoạn con thì xếp chúng giảm dần theo bảng chữ cái.

Ví dụ:

Input:

TATATGCTCT
3
0 9
5 9
5 5

Output:

TCAG
TCG
G

Cách làm 1: Giải trực tiếp

Tại bước này, việc chúng ta cần là xây dựng mmm tại mỗi truy vấn. Để đơn giản nhất, ta xây dựng trực tiếp map bằng cách trích xuất đoạn con từ xâu S.

string s; int q;
map<char, int> mq;
void truyvan(int l, int r){
	//"thay mới" map m cho mỗi truy vấn để không bị cộng dồn, 
	//tức là kết quả của truy vấn trước không gây ảnh hưởng
	//đến map m ở truy vấn sau
	m.clear();
	for (int i = l; i <= r; i++){
		//Để xây dựng map m có thể sử dụng tính chất của map
		//Trong C++, khi một map được gọi với một khóa chưa có
		//thì giá trị của khóa đó sẽ được cho là 0 với các dạng số 
		//như int, long long, double, ...; 
		//và vector/map/string... rỗng với các kiểu dữ liệu tương đương
		//Tính năng này được gọi là zero-initilization
		
		m[s[i]] += 1;
	}
	
	//xây dựng xâu X
}

Độ phức tạp: O(Q∗N)O(Q * N)O(Q∗N) do mỗi truy vấn cần O(N)O(N)O(N) thao tác, cộng thêm việc xây dựng xâu XXX mất thêm O(N)O(N)O(N) thao tác nữa, tất cả những điều này lặp lại QQQ lần. Với điều kiện của đề bài thì cách này rõ ràng không đủ thời gian. Vì vậy, ta cần tìm một cách khác.

Ta có thể chứng minh điều tương tự: với một xâu S[0..N−1]S[0..N-1]S[0..N−1] độ dài bất kỳ và mọi truy vấn có l=p,r=ql = p, r = ql=p,r=q sao chó 0≤p≤q≤n−10 \leq p \leq q \leq n - 10≤p≤q≤n−1, chênh lệch các giá trị giữa l=0,r=ql = 0, r = ql=0,r=q và l=0,r=p−1l = 0, r = p - 1l=0,r=p−1 (với p≥1p \geq 1p≥1, còn nếu p=0p = 0p=0 thì kết quả sẽ giống bảng trên) là giống nhau. Ta dễ dàng thấy M(A)+M(B)=M(A+B)M(A) + M(B) = M(A + B)M(A)+M(B)=M(A+B) với A,BA, BA,B là các xâu, A+BA + BA+B là xâu ghép nối của AAA và BBB và với T(S,l,r)=M(S[l..r])T(S, l, r) = M(S[l..r])T(S,l,r)=M(S[l..r]):

M(S,0,r)=M(S[o..l−1])+M(S[l..r])→T(S,l,r)=T(S,0,r)−T(S,0,l−1) M(S, 0, r) = M(S[o..l - 1]) + M(S[l..r]) \\ \rightarrow T(S, l, r) = T(S, 0, r) - T(S, 0, l - 1) M(S,0,r)=M(S[o..l−1])+M(S[l..r])→T(S,l,r)=T(S,0,r)−T(S,0,l−1)

vì T(S,l,r)T(S, l, r)T(S,l,r) là map, nên kết quả của phép trừ trên được định nghĩa khi 2 map có cùng bộ khóa, và map kết quả có khóa thứ i=i =i= giá trị khóa đó ở map thứ nhất −-− giá trị khóa đó ở map thứ hai. Như vậy, ta chỉ cần xây dựng T(S,0,r)T(S, 0, r)T(S,0,r) tại mỗi giá trị 0≤r≤n−10 \leq r \leq n - 10≤r≤n−1. Để tránh tính lại các giá trị nhiều lần, ta có thể lưu chúng vào một mảng, với giá trị thứ iii tương đương với T(S,0,i)T(S, 0, i)T(S,0,i)

const int maxn = 1e5 + 1;
map<char, int> m[maxn], mq;
string s; int q;

void truyvan(int l, int r){
	mq.clear();
	for (char c = 'A'; c <= 'Z'; c++){
		if (l > 0) mq[c] = m[r][c] - m[l - 1][c];
		else mq[c] = m[r][c];
	}
	
	//xây dựng xâu X dựa trên mq
}

int main(){
	//xử lý input / output
	//lưu ý đề bài không cho độ dài của S
	int n = s.length();
	for (int i = 0; i < n; i++){
		for (int j = 0; j < i; j++){

			m[i][j] += 1; 
		}
	}
	//xử lý truy vấn
}

Cách làm 3: Tối ưu hóa cách giải gián tiếp

Vậy ta cần tối ưu hóa cách làm này như thế nào? Trước hết, ta cần quay lại bảng 1, và nhận thấy rằng mỗi khi giá trị rrr tăng lên 1 thì giữa T(S,0,r)T(S, 0, r)T(S,0,r) và T(S,0,r+1)T(S, 0, r + 1)T(S,0,r+1) có đúng 1 khóa có giá trị tăng lên 1. Ta cũng dễ dàng giải thích điều đó.

T(S,0,r+1)=T(S,0,r)+T(S,r+1,r+1) T(S, 0, r + 1) = T(S, 0, r) + T(S, r + 1, r + 1) T(S,0,r+1)=T(S,0,r)+T(S,r+1,r+1)

và lưu ý rằng T(S,r+1,r+1)=M(S[r+1..r+1])=M(S[r+1])T(S, r + 1, r + 1) = M(S[r + 1..r + 1]) = M(S[r + 1])T(S,r+1,r+1)=M(S[r+1..r+1])=M(S[r+1]) và sẽ là 1 map có duy nhất 1 khóa có giá trị bằng 111, còn lại tất cả các khóa có giá trị bằng 000. Như vậy, ta có thể thay đôi đoạn code tính các giá trị M(S[0..r])M(S[0..r])M(S[0..r]) ở trên bằng cách lưu ý thay đổi nhỏ này.
